## Стратегия: Отдельные страницы для треков

### Структура URL:
```
/ru/track/japanese-lofi-music
/ru/track/lofi-music-for-studying
/ru/track/rainy-lofi-music
/en/track/japanese-lofi-music
/en/track/lofi-music-for-studying
```

### Что должно быть на странице трека:

1. **Заголовок H1** с ключевым запросом
2. **Описание трека** (200-300 слов) с ключевыми словами
3. **Плеер** с этим треком (автозапуск)
4. **Связанные треки** (похожие по тематике)
5. **Schema.org** для MusicRecording
6. **Мета-теги** оптимизированные под запрос

### Пример структуры:

```html
/track/japanese-lofi-music.html
- Title: "Japanese Lofi Music - Listen Online Free | Lofi Radio"
- H1: "Japanese Lofi Music - Relaxing Tokyo Beats"
- Description: "Listen to Japanese lofi music online. Relaxing Tokyo beats perfect for studying, work or relaxation. Free lofi radio without ads."
- Keywords: "japanese lofi music, tokyo lofi beats, japan lofi, cute japanese lofi"
```

## Альтернатива: Блог статьи

### Структура:
```
/blog/lofi-music-for-studying.html
/blog/rainy-lofi-music.html
/blog/japanese-lofi-music.html
```

### Преимущества блога:
- Больше контента для SEO
- Можно добавить больше ключевых слов
- Можно делиться в соцсетях
- Больше времени на сайте

### Структура статьи:
1. Заголовок с ключевым запросом
2. Введение (100-150 слов)
3. Описание темы (300-500 слов)
4. Встроенный плеер с релевантными треками
5. Заключение
6. Связанные статьи

## Рекомендации

### Приоритет 1: Блог статьи (лучше для SEO)
- Создать 5-7 статей под популярные запросы
- Каждая статья 500-800 слов
- Встроить плеер с релевантными треками
- Добавить внутренние ссылки

### Приоритет 2: Страницы треков (если нужны отдельные страницы)
- Создать страницы для топ-5 треков
- Оптимизировать под узкие запросы
- Добавить Schema.org для каждого трека

### Приоритет 3: Категории/теги
- Группировать треки по темам
- Создать страницы категорий:
  - /ru/category/study-music.html
  - /ru/category/sleep-music.html
  - /ru/category/rainy-music.html
